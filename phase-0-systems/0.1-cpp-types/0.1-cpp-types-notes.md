Working code and observations for topic 0.1.

## Exercise 1 — Types and value semantics

Source: `ex1.cpp`.

### 1.1 — sizeof of types, pointers, and a struct
Prints the size in bytes of `int`, `char`, `bool`, `double`, `int*`, `double*`,
and a small `struct { int x; char c; }`.

### 1.2 — Observations
- **What `int*` and `double*` have in common:** both are pointers, that is,
  memory addresses, and every memory address takes the same space (8 bytes on
  64-bit, 4 bytes on 32-bit), regardless of what they point to.
- **Why the struct's size is not exactly the sum of its members:** members are
  stored in blocks; the total size is rounded up to a multiple of the alignment,
  which determines the block size.

### 1.3 — Value semantics (pass by value)
`addOne(int n)` does `n += 1` and returns nothing. Calling `addOne(a)` does not
change `a`, because inside the function `n` is not `a` itself but a copy of it;
the function only modifies its copy.

## New concepts

- **bookkeeping** — the work the allocator does to find a free block of memory when you allocate on the heap (e.g. with `new`).
- **padding** — filler bytes the compiler inserts between or after struct members so that each member starts at a properly aligned address (and the struct's total size is a multiple of its alignment), which lets the CPU read the members efficiently. Where the padding lands depends on the order of the members:

```
struct MyStruct {
    int x;
    char c;
};

offset:  0    1    2    3    4    5    6    7
        [------- int x -------][ c ][-- padding --]
```

```
struct MyStruct2 {
    char c;
    int x;
};

offset:  0    1    2    3    4    5    6    7
        [ c ][--- padding ---][------- int x -------]
```

Same total size (8 bytes), but the padding moves: at the end (after `c`) in the first; in the middle in the second, so that `int x` still starts at a 4-aligned offset.

- **pointer size** — `int*` and `double*` are memory addresses; on a 64-bit build an address takes 8 bytes, on 32-bit it takes 4 — regardless of what they point to.
